# LAVAD 视频异常检测系统 - 项目技术文档

## 一、项目概述

### 1.1 项目简介

本项目是对 CVPR 2024 论文《**LAVAD: Language-guided Video Anomaly Detection**》的完整复现与工程化部署。系统基于大语言模型（LLM）驱动的方法实现视频异常检测，无需在目标数据集上进行训练，仅使用预训练模型完成零样本（zero-shot）异常检测。

### 1.2 核心特性

- **零样本异常检测**: 无需在监控数据集上训练，直接使用预训练模型推理
- **语言驱动**: 利用图像描述生成（BLIP-2）、跨模态检索（ImageBind）、LLM推理（Qwen-7B）构建完整pipeline
- **多模型集成**: 使用5种BLIP-2模型提高描述多样性，结合ImageBind进行跨模态匹配
- **8步流程**: 从帧提取到最终评估的完整标准化流程
- **Web界面**: Streamlit前端 + FastAPI后端的完整Web服务

### 1.3 预期应用场景

监控视频实时/离线异常检测、公共安全预警、智慧城市建设等。

## 五、核心算法与数据流

### 5.1 完整数据流

```
输入视频 (.mp4/.avi)
    │
    ▼ Step 1: 帧提取
frame_0.jpg, frame_1.jpg, ..., frame_N.jpg  (路径列表)
    │
    ▼ Step 2: BLIP-2 Caption生成 (5个模型顺序执行)
{model_1: {clip_0: "a man walking...", clip_1: "a car passing..."}, ...}
    │
    ▼ Step 3: ImageBind文本索引 (captions → text embeddings → FAISS)
faiss_index (IndexFlatIP, dim=1024)
    │
    ▼ Step 4: 跨模态Caption清洗 (vision features → search faiss → filter)
cleaned_captions[model][clip_idx]["0"] = {"caption": "...", "similarity": 0.xx}
    │
    ▼ Step 5: LLM摘要与评分 (Qwen-7B推理)
summaries[i] = "A group of people gathered..."
raw_scores[i] = 0.6
    │
    ▼ Step 6: 摘要索引创建 (summaries → embeddings → FAISS)
summary_index (IndexFlatIP, dim=1024)
    │
    ▼ Step 7: 多邻居分数细化 (vision features → search summary_index → weighted avg)
refined_scores[i] = Σ(neighbor_score × exp(similarity)) / Σ(exp(similarity))
    │
    ▼ Step 8: 最终评估 (refined_scores vs threshold)
anomaly_frames = [f for f in frames if refined_scores[f] > max(mean, 0.45)]
    │
    ▼ 输出
DetectionResult + ZIP导出
```

### 5.2 关键算法详解

#### 5.2.1 自适应帧间隔

```python
if total_frames < 150:
    interval = FRAME_INTERVAL_SHORT  # 4帧间隔 (短视频)
elif total_frames < 300:
    interval = FRAME_INTERVAL_MEDIUM # 8帧间隔 (中视频)
else:
    interval = FRAME_INTERVAL        # 16帧间隔 (长视频)
```

#### 5.2.2 跨模态Caption清洗

对每个clip：
1. 使用ImageBind提取视觉特征向量（vision embedding）
2. 在FAISS文本索引中检索最相似的caption（cosine similarity via IndexFlatIP）
3. 保留相似度最高的caption作为该clip的"清洗后描述"

```python
clip_subsample_paths = uniform_temporal_subsample(clip_frame_paths, NUM_SAMPLES)
vision_inputs = data.load_and_transform_vision_data(clip_subsample_paths, "cuda:0")
vision_embeddings = self.imagebind_model({ModalityType.VISION: vision_inputs})
vision_vectors = vision_embeddings[ModalityType.VISION].cpu().numpy()
faiss.normalize_L2(vision_vectors)
D, I = text_index.search(vision_vectors, NUM_NEIGHBORS_STEP3)
# 按模型分组保留最相似的caption
cleaned_captions[model_path][str(clip_idx)][str(ci)] = {
    "caption": cap,
    "similarity": 1.0
}
```

#### 5.2.3 多邻居分数细化

对每个clip：
1. 使用ImageBind提取视觉特征
2. 在摘要FAISS索引中检索k=10个最相似邻居
3. 使用指数加权平均：`refined = Σ(score_i × exp(sim_i)) / Σ(exp(sim_i))`

```python
D, I = summary_index.search(vision_vector, NUM_NEIGHBORS_STEP6)
exp_weights = np.exp(frame_similarities_arr)
frame_weights = exp_weights / np.sum(exp_weights)
refined_score = float(np.sum(frame_scores_arr * frame_weights))
refined_score = max(0.0, min(1.0, refined_score))
```

#### 5.2.4 异常判定阈值

```python
mean_score = np.mean(refined_scores)
std_score = np.std(refined_scores)

if len(scores) < MIN_CLIPS_FOR_STATS:
    threshold = 0.45          # 样本不足用固定阈值
elif std_score < 0.005:
    # 评分坍缩保护：回退到Step 5原始评分
    print(f"WARNING: Step 6 collapsed all scores to std={std_score:.6f}")
    final_scores = raw_scores
    threshold = max(np.mean(raw_scores), 0.45)
else:
    threshold = max(mean_score, 0.45)
```

## 六、显存管理与GPU优化

### 6.1 三级显存管理

```
Level 1: BLIP-2单模型加载
  ┌─────────────────────────────────┐
  │ for model in BLIP2_MODELS:       │
  │   load_model(model)              │
  │   process_all_frames(model)      │
  │   unload_model(model)            │
  │   torch.cuda.empty_cache()       │
  └─────────────────────────────────┘

Level 2: ImageBind/Qwen临时卸载
  ├─ _temp_unload_for_blip2()
  │   └─ imagebind.to('cpu'), qwen.to('cpu')
  ├─ BLIP-2 processing...
  └─ _restore_after_blip2()
      └─ imagebind.to('cuda:0'), qwen.to('cuda:1')

Level 3: T5-onnx 端到端100B以内占用
```

### 6.2 GPU分配策略

| 模型 | GPU分配 | 原因 |
|------|---------|------|
| ImageBind | GPU 0 (固定) | ~3.4GB，视觉特征提取频繁 |
| Qwen-7B-Chat | GPU 1 (固定) | ~15GB，需要独立GPU空间 |
| BLIP-2 × 5 | GPU 0 (临时) | 每个约7~12GB，依次加载 |

## 七、Prompt工程

### 7.1 模板体系

系统采用三层Prompt模板：

1. **System Prompt**: 设定LLM角色（场景分析者 + 评分者）
2. **User Prompt**: 包含具体任务指令
3. **Format Prompt**: 约束输出格式（Python list，0~1分数）

### 7.2 Qwen Chat Template

```python
self.qwen_tokenizer.chat_template = """\
{% for message in messages %}<|im_start|>{{ message['role'] }}
{{ message['content'] }}<|im_end|>
{% endfor %}<|im_start|>assistant"""
```

### 7.3 推理参数

- `temperature = 0.6` — 在多样性和一致性间平衡
- `top_p = 0.9` — 核采样保留90%概率质量
- `max_new_tokens = 256` — 限制最大生成长度
